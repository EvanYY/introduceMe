  • 页面组成：
  • 1.首页( 广告轮播 + 商品数据 + 商品分类 + 错误拦截)
  • 2.商品列表页( 商品类别过滤 + 价格过滤 + 价格排序 + 懒加载 + 滚动分页 + 加入购物车 + 跳转商品详情页 + 登录拦截 + 消息提示)
  • 3.商品详情页 ( 猜你喜欢模块 + 商品信息加载 + 立即支付 + 加入购物车 + 商品数目变化 + 错误信息提示 + hash传参)
  • 4.购物车页面：( 购物商品列表获取 + 全选/单选 + 编辑数量 + 删除单条数据 + 批量删除 + 信息不完整拦截 + 购物车信息状态进行提示)
  • 5.地址信息: ( 地址新增 + 删除 + 设为默认地址[进行排序，把默认地址放在第一个] + 数量过滤 + 信息确认进行订单信息核对页面)
  • 6. 订单信息：(查询购物信息 + 费用详情 + 信息核对 + 确定支付，注意哦...详情页的立即支付与购物车传来的立即支付判断规则不同)
  • 7. 订单创建成功页面获取订单信息 进行下一步操作；
  • 8.头部公共组件(最让我痛苦的组件)： (用户登录 + 注册校验原生的哦 + 购车页面拦截 + 用户名是否注册校验)
  • 总结： 页面信息传递通过vuex，hash和基本的父子组件通信，前后端的难点是业务逻辑的判断
